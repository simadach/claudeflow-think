## 🔴 見落としている重大なリスク

- [ ] #001 **プラットフォーム選択が将来の CPU 交換可能性を決定してしまうかもしれません**
  Ryzen 9 7900X（AM5）と Core i7-14700K（LGA1700）を並べているが、これはマザーボードごと変わるアーキテクチャ選択。LGA1700 は現世代が最後（次世代は LGA1851）であり、将来「CPU だけ交換」したい場合 AM5 の方が数年先まで互換性が続く可能性がある。どちらを選ぶかで 3〜5年後の選択肢が変わる。

- [ ] #002 **BTO を選んだ場合、将来の GPU 交換が制約を受けるかもしれません**
  BTO のケースや電源はカスタムサイズ・OEM 仕様のことがあり、市販の大型 GPU（次世代 RTX など）に交換しようとしたときに収まらない・コネクタが合わないケースがある。「数年後に GPU だけ交換」という目標と BTO は相性が悪い可能性がある。

## 🟡 検討すべきトレードオフ・代替案

- [ ] #003 **RTX 40xx 系は既に「前世代」になっているかもしれません**
  2026年6月時点では RTX 5000シリーズが発売済みの可能性が高い。RTX 5070 Ti あたりが 4070 Ti Super に近い価格帯で上回るなら、今 4070 Ti Super を買うのは割高な選択になるかもしれない。現在の新品・中古の実売価格と RTX 50xx の立ち位置を確認したか？

- [ ] #004 **「省スペース」の定義がまだ曖昧で、購入後に後悔するかもしれません**
  280mm AIO を搭載できる mATX ケースは高さ 30〜45cm になることが多い（例: Fractal Design Meshify 2 Compact は高さ 41cm）。Mac mini M4 と並べた場合、「圧迫感のない見た目」という期待と現実が乖離するかもしれない。許容できる最大サイズ（高さ・奥行き）を cm で決めてから逆引きでケースを絞ると後悔が少ない。

- [ ] #005 **Blender の大規模シーンでは GPU VRAM が先に詰まるかもしれません**
  64GB RAM を想定しているということは重いシーンを扱う可能性がある。RTX 4070 Ti Super の VRAM は 16GB であり、GPU レンダリング（OptiX）ではシーン全体が VRAM に乗り切らないと CPU レンダリングにフォールバックする。「Blender を GPU で速く回す」目標に対して VRAM 量が制約になるシナリオを具体的にイメージしているか？

## ❓ 確認すべき前提・仮定

- [ ] #006 **3用途が本当に 1台に同時に要求されるのか、時間的に分離できるのか確認が必要かもしれません**
  「ゲームをしながら Blender をバックグラウンドでレンダリング」は GPU が 1枚しかない構成では実質不可能（どちらかが著しく遅くなる）。用途の同時使用シナリオを明確にしないと、「1台で全部」の意味が曖昧なまま購入してしまうかもしれない。

- [ ] #007 **Mac mini M4 と Windows PC の役割分担が購入後に曖昧になるかもしれません**
  Apple Silicon の Blender（Metal バックエンド）は M4 でも実用的に動く。Windows PC 購入後に「Fusion 360 はどっちでやる？Blender は？」という問いに具体的な答えがないと、片方が稼働率ゼロになる可能性がある。2台持ちの明確な役割分担を先に決めておいた方がよいかもしれない。

- [ ] #008 **高負荷時の消費電力が HEMS に影響するか確認が必要かもしれません**
  RTX 4070 Ti Super（TDP 285W）＋ Ryzen 9 7900X（TDP 170W）構成の高負荷時は電源から 550〜650W 引く可能性がある。Blender のレンダリングを昼間に回すと太陽光・蓄電池の余剰電力を食い潰すかもしれない。あるいは夜間 EV ナイトタイム（23.43円/kWh）に回す運用を想定しているか？年間電気代への影響試算はしているか？
