## フォームファクター選定の根拠

mini-ITX を選んだ理由：
- 「mini-ITX 程度が理想」の省スペース要件を優先
- 単一 GPU のみ（PCIe スロット追加不要）
- 同時 3D 計算なし（時間分離で使い分け）→ 拡張カード不要

当初の「ITX は拡張性が不安」は以下の理由で解消：
- GPU は PCIe x16 スロット 1本で十分（将来交換も同じスロット）
- M.2 2本（マザーボード）→ 2TB + 拡張スロット確保可能
- AM5 プラットフォームでの CPU 換装（マザーボード流用）は引き続き可能

## 背景・動機

- 3Dプリンター用のモデルを自分で設計したい（Fusion 360、Blender 等）
- Blenderで3DCGモデリング・ライティング・レンダリングをやりたい（軽〜中規模シーン）
- 最新AAAタイトルを 1440p / 60fps 以上で遊びたい
- Mac mini M4 はすでに持っているが、それとは別にWindowsゲーミング兼クリエイター機が欲しい
- 机上スペースが限られているため、mini-ITX レベルのコンパクト機が理想

---

## 期待する結果・価値

- Blender のレンダリングが実用的な速度で完了する
- AAA ゲームが 1440p / 60fps 以上で動く
- 3Dプリンター用モデル（Fusion 360 等）の操作が快適
- デスクに置いても圧迫感がない見た目（Lian Li A4-H2O はスリムでモダン）
- 数年後に GPU を PCIe x16 スロットで交換できる

---

## 前提条件

- Windows 環境であること（ゲームのため）
- **用途は時間分離が前提**: ゲーム・Blender レンダリング・3D モデリングを同時実行はしない
- BTO ではなく**自作**を選択する（mini-ITX BTO は選択肢が極めて限られるため）
- Mac mini M4 との役割分担を明確にする（下記「役割分担方針」を参照）
- **熱・騒音の体験を理解した上で選択する**: CPU（120W）+ GPU（280W）= 400W 前後が 16L に集中するため、高負荷時はタワー型より騒音が目立つ。ただし thermal throttling は現実的には起きない。部屋にエアコンがあれば夏季も実用範囲内。この特性を許容した上で SFF を選択している。

---

## Mac mini M4 との役割分担方針

| 用途 | 担当機 | 理由 |
|------|--------|------|
| AAAゲーミング | Windows PC 専用 | macOS では動作しないタイトル多数 |
| Blender レンダリング | Windows PC（GPU/OptiX） | CUDA/OptiX が速い |
| Blender 軽量モデリング | どちらでも可 | M4 も十分実用的 |
| Fusion 360（3Dプリンター用） | Windows PC 優先 | Windows 版が安定・機能豊富 |
| 日常開発・HEMS作業 | Mac mini M4 継続 | 変更なし |

→ **「Windows PC はゲームと Blender レンダリング専用機」**

---

## 意識的に考慮したが却下した観点

- **大規模 Blender シーンへの対応**: 使わないためスルー（VRAM 16GB で十分）
- **HEMS への電力影響**: 蓄電池でカバー可能なため考慮しない
- **BTO**: mini-ITX BTO の選択肢が少なく、自作より割高かつ GPU 交換制約のリスクあり

| RAM を 32GB (2×16GB) に削減 | △約4万円 | Blender 大規模シーン不可（軽シーン専用なら問題なし） |
