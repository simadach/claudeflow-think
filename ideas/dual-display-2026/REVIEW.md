## 🔴 購入前に確認すべき重大リスク

- [x] #001 **Anker PowerExpand 5-in-1 TB4 Dock が Dual 4K@60Hz 同時出力に対応しているか未確認かもしれません** ✅ 反映済み
  TB4 ドックには「映像出力が1ポートのみ」または「TB4 スルーポート × 1 + HDMI × 1」という構成のモデルが多く存在します。「5-in-1」という名称から USB-C 映像出力 × 2 本が取れると思いがちですが、Dual 4K@60Hz を1本の TB4 ケーブルで実現するには「Dual Stream (DSC)」対応が必要です。¥24,980 の価格帯に合うモデルが Dual 4K@60Hz に本当に対応しているか、Anker 公式スペックの「Dual 4K 対応」表記を購入前に確認することをお勧めします。
  🔴 リスク: 高（対応していなければ構成の根幹が崩れる）
  > 返答:

- [x] #002 **Auto Input Detection が「USB-C ケーブル挿しっぱなし・映像信号なし」を正しく検出できるか未確認かもしれません** ✅ 反映済み
  Mac がスリープ/シャットダウンしても、Anker Dock 経由の USB-C ケーブルは PD 給電（5V）を継続するモデルがあります。モニター側から見ると「ケーブルは接続中だが映像信号なし」という状態になり、AID が「PC が消えた」と判断できず切り替えが発生しないかもしれません。特に LG 32U720A-B は 2025年6月発売の新モデルでユーザーレビューが少なく、USB-C 接続での AID 挙動が実証されていない可能性があります。
  🔴 リスク: 中〜高（切り替えが自動化されない場合、手動切り替えが日常になる）
  > 返答:

- [x] #003 **Anker Dock の出力端子が USB-C (DP Alt Mode) × 2 本であることを確認できていないかもしれません** ✅ 反映済み
  「Anker Dock → USB-C × 2 → 両モニター」という接続図が前提になっていますが、Anker TB4 ドックの映像出力が「TB4 × 1 + HDMI × 1」の場合、HDMI → 32UL750-W は HDMI 2.0（4K@60Hz 対応）なので問題ないものの、接続構成が変わります。購入前に Anker 公式の接続ポート図（どのポートが映像出力か）を確認しておくと安心かもしれません。
  🔴 リスク: 中（構成変更が必要になる可能性）
  > 返答:

## 🟡 検討すべきトレードオフ・代替案

- [x] #004 **Windows WS 購入後、DP MST Hub が不要になる可能性があるかもしれません** ✅ 反映済み
  pc-build-2026 の GPU（RTX 5070）は通常 DisplayPort × 3 + HDMI × 1 を持ちます。この場合、MST Hub なしで2台のモニターに直接 DP × 2 接続が可能です。Windows WS 購入前に DP MST Hub を買ってしまうと、結果的に不要になるかもしれません。WS 購入まで DP MST Hub の購入を保留し、GPU のポート構成を確認してから買う方が無駄がないかもしれません。
  🟡 リスク: 低（無駄な出費で済む範囲）
  > 返答:

- [x] #005 **Anker Dock 周辺のケーブルが「デスクをすっきり」の目標に反するかもしれません** ✅ 反映済み
  「各 PC からケーブル 1 本」は達成されますが、Anker Dock 自体が DC アダプター（電源ケーブル）を必要とし、Dock → モニター1/モニター2 の出力ケーブルも2本追加されます。Mac 側だけで「Mac → Dock」「Dock → モニター1」「Dock → モニター2」「Dock 電源」の合計4本のケーブルがデスク上に存在するかもしれません。ケーブルトレー等の配策を事前に検討しておくと良いかもしれません。
  🟡 リスク: 低（体験の期待値調整）
  > 返答:

- [x] #006 **Windows → DP MST Hub の接続で、再起動・BIOS 画面が片方のモニターにしか映らないかもしれません** ✅ 反映済み
  MST Hub 経由の接続では、Windows 起動時の BIOS/UEFI 画面やロック画面が「メインモニターのみ」に表示されることがあります。MST Hub は OS が起動してドライバーが読み込まれてから完全に機能するため、それ以前の画面は1台にしか映らないかもしれません。デュアル画面を前提に作業する場合、起動直後に片方が真っ暗な時間があることを想定しておくと良いかもしれません。
  🟡 リスク: 低（操作上は問題ないが体験として気になる可能性）
  > 返答:

## ❓ 確認すべき前提・仮定

- [x] #007 **LG 32U720A-B の AID（Auto Input Switch）が公式スペックに明記されているか確認されていないかもしれません** ✅ 反映済み
  LG の一部モニターは「Auto Input Switch」を明記していますが、32U720A-B は 2025年6月発売の新モデルです。LG 公式の製品ページで AID（または Auto Input Detection/Auto Input Switch）の対応が明示されているか、購入前に確認することをお勧めします。仕様表になければ、ファームウェアで有効化できるか（LG OnScreen Control）も確認ポイントかもしれません。
  > 返答:

- [x] #008 **両 PC が同時起動している際に AID が意図しない切り替えを行うかもしれません** ✅ 反映済み
  Mac を使用中に Windows を起動・シャットダウンする場合、Windows の起動シーケンスで映像信号が発生・消滅し、AID が「新しい信号を検出」→「切り替え」と判断してモニターが突然 Windows 画面に切り替わるかもしれません。LG の AID は「最後にアクティブだった入力を優先する」設定が多いですが、「新しい信号を検出したら自動切り替え」のデフォルト設定は使用体験に影響します。AID の優先度設定（Manual/Auto）を購入後すぐに確認・調整することをお勧めします。

## 💭 言語化されていない感覚・体験

- [x] #009 **「自動切り替えが遅い」「2台が同時に切り替わらない」ストレスが蓄積するかもしれません** ✅ 反映済み
  AID の切り替え時間は一般的に 5〜15 秒で、かつ2台のモニターが同じタイミングで切り替わらないことがあります。Mac → Windows に切り替えようとするたびに「モニター1 は切り替わったが、モニター2 はまだ Mac の画面を表示している」という状態が数秒続くかもしれません。最終的に「AID を無効にして、モニター本体のジョイスティックボタンで手動切り替え」の方が快適と感じる可能性があります。
  > 返答:

- [x] #010 **Windows で HDR を「常に ON」にすると、モニター間の表示差が想像以上に気になるかもしれません** ✅ 対象外（返答: 常時 HDR にはしない。対応コンテンツを見るときだけオン）
  「SDR 主体なら許容範囲」と判断されていますが、ゲーム中は Windows が HDR モードを自動的に有効化する場合があります。HDR 有効時、32UL750-W（DisplayHDR 600・1000nit）と 32U720A-B（HDR10 のみ・400nit 程度）では明暗の差が大きく、ゲームを広い視野でプレイするときに「左右のモニターの色味・明るさが全然違う」と感じるかもしれません。ゲーム用途でデュアルモニターを前提にするなら、非アクティブ側のモニターの HDR を無効にする設定を検討しておくと良いかもしれません。
  > 返答:常時hdrにはしない。対応こんてんつを見るときだけオンにする
