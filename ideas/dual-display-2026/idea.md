**作成日**: 2026-06-24
**最終更新**: 2026-06-25（DisplayLink再除外・A83B5/直結2本に絞込み）
**ステータス**: 構成検討中

---

## 核心

Mac Mini と将来購入予定の Windows WS を、各1本のケーブルで共有デュアルディスプレイに接続する。モニターはAuto Input Detectionで自動切り替え。
**ケーブル構成要件**: 各 PC → ドック 1本・ドック → 各モニター 1本。分岐器・切り替え器などの中間機器は使わない。

---

## 背景・動機

- Mac Mini M4（メイン）+ Windows WS（予定）の2台体制に移行する
- 既存モニター LG 32UL750-W が1台あり、もう1台追加してデュアル化したい
- デスクをケーブルで汚したくない。各コンピューターからの出力は1本に絞りたい
- モニター2台を物理的にリンク（デイジーチェーン等）させたい

## 既知の懸念・トレードオフ

1. **HDR品質の差**：32UL750-W（DisplayHDR 600）と32U720A-B（HDR10のみ）でHDR品質が揃わない。HDRは対応コンテンツを見るときだけONにする運用のため、常時の表示差は問題にならないと判断。

2. **Auto Input Detection依存**：切り替えはモニター側機能に依存。応答速度は実機確認が必要。AIDの切り替えには5〜15秒かかり、2台のモニターが同時に切り替わらない場合がある。遅延が気になる場合は手動切り替えに変更する運用も想定。

3. **DP MSTハブ品質リスク（Windows側）**：安価品は4K/60Hz安定性にばらつきあり。BIOS/UEFIやロック画面はMST Hub非対応のため起動直後は片方のモニターにしか映らない場合がある。RTX 5070でDP×3確保できれば回避。

4. **Windows WS未購入**：GPUのポート構成により接続方式が変わる。WS購入後に確認が必要。
   - WS接続ポート案：32U720A-B → Windows は HDMI 2.1 または USB-C 入力、32UL750-W → Windows は HDMI port 2 または DP 1.4 入力。WS購入時にGPUの出力端子と照合が必要。

5. **両PC同時稼働時の意図しないAID切り替え**：Mac使用中にWindowsを起動すると映像信号をAIDが検出して切り替わる可能性。AID設定を「Manual」または「Last Input」に変更することで回避できる場合が多い。購入後すぐに設定確認・調整が必要。

6. **「1本縛り」とコストのトレードオフ（未解決）**：Mac Mini直結2本なら追加コスト0だが「1本縛り」を諦める。TB4/TB5ドック購入は¥49,990〜58,300の追加コストで「1本縛り」を達成する。この差額が体験価値に見合うかが未回答。

7. **DP接続時の32U720A-B内蔵USBハブ不可**：32U720A-B のUSB-C端子は「映像入力＋USBハブ＋PD 90W給電」兼用の upstream ポート。Anker Prime経由でDP 1.4入力を使う構成では、モニター内蔵USBハブ機能はMac Miniから使えない。Mac Mini本体・Anker PrimeのUSBポートで代替可能なため実害は少ない。

8. **デザイン選好は長期満足度に影響する**：毎日長時間作業するデスク環境において、機器の見た目に対する美的満足感は気分・生産性に実際に影響する。スペック比較では数値化されないが、購入後の継続的な使用満足度に関わる重要な要素。TB4/TB5ドックで見た目の好みを完全に満たすものがない場合、この点を意識して選定する。

## 前提条件

- Windows WS購入時のGPUがDP 1.2以上のMST対応（現行GeForce/Radeonはほぼ全対応）
- Mac Mini M4のThunderbolt 4ポートでUSB-C映像出力2画面が可能（仕様上対応）
- LG 32U720A-BのAuto Input DetectionがTB4/USB-C信号消失を正しく検出できる

## 意識的に考慮したが却下した観点

**Thunderboltデイジーチェーン（1本で2台）**：
LG 32UL750-WにはThunderbolt入力がないため非対応と判明。断念。

**32UL950-W / 32UN650-W / 32UN950-W 中古**：
デザインを揃えるため検討。中古市場調査で在庫がほぼ存在せず断念。

**Dell U2723DE（Thunderbolt hub monitor）×2**：
TB4 + DP MST両対応だが¥80,000〜100,000/台で予算超過。断念。

**32U631A-B**：
安価（¥30,250）だが4K非対応（WQHD）・USB-C PD 15Wのみで不十分。

**A83B35A1（DisplayLink）**：
見た目が最も好み・LAN・SDカード・本体ディスプレイが魅力的だったが、以下の理由でDisplayLinkを完全除外した。

- **HDCP制限は全モニターに波及**：「DisplayLinkモニター以外は影響なし」という期待は外れる可能性が高い。macOSの HDCP実装では「DisplayLink接続デバイスがシステムに存在する」だけで、直接接続のモニターでもNetflix・Apple TV+の再生が止まる。「Monitor 1（直結）でNetflixを見る」という回避策は機能しない。
- **スリープ問題はゼロにならない**：Mac Miniの常時起動運用でも、macOSアップデートのたびにDisplayLink Managerとの相性問題が再発するリスクがある（1〜2週間動作不可の事例あり）。
- **Mac Desktop環境の使用報告が少ない**：A83B35A1は比較的新しく、Mac Mini環境での「問題なし」を示すエビデンスが現状存在しない。
- **DisplayLinkの摩擦は「慣れ」で解決しない**：DRM再生不可の想起コスト・OSアップデート後の障害・DisplayLink Manager常駐の心理的負荷は、使い続けるほど蓄積する種類の摩擦。継続的な摩擦を嫌うスタイルと相性が悪い。
- **¥11,281 差額はリスク代として許容**：A83B35A1（¥38,709）vs A83B5（¥49,990）の差額¥11,281は、デザインよりも「構成の安定」を優先する判断により、安定性への対価として許容する（2026-06-25確認）。

**A83B6 Prime（Dual Display 160W、¥29,990）**：
DisplayLinkなし・LAN搭載・¥29,990と魅力的だったが、
映像出力がDP Alt Mode → MST分岐 → HDMI×2という方式のため、
macOSがMST非対応であることが判明。Mac Mini M4では1画面しか映らない（またはミラーリングのみ）。
→ 完全除外。macOS環境では使い物にならない。

**Anker Prime A83B5（TB5、¥49,990）と1本縛りの費用対効果**：
TB5ドックはMac Mini M4（TB4）と後方互換で動作し、DisplayLinkなしでデュアル4K可能。
LAN・USB-A・SDカードも充実。ただし¥49,990は高価で、TB5の帯域はMac Mini M4では活かせない（TB4の最大40Gbpsに制限される）。現在の4K@60Hzデュアル＋通常USB周辺機器であれば帯域は十分。
「Mac Miniから直接2本接続すれば¥0」という事実と比較して、¥49,990の価値があるかは引き続き未回答。
→ 現在の有力候補（DisplayLink除外後の第一選択肢）。

**⚠️ メルカリ中古でAnker Primeを探す場合のモデル番号確認**：
Anker Prime には複数モデルが存在する。主な2モデル：**A83B5**（TB5・14-in-1・8K対応）と**A83B6**（Dual Display・160W）でポート構成が大きく異なる。特にDP 2.1出力・TB5ダウンストリームポート数が変わるため、出品ページや現物でモデル番号を必ず確認すること。A83B6はmacOS MST非対応のためデュアル不可（除外済み）。

**KVMスイッチ案**：
Mac・Windowsの切り替えをKVMスイッチ1台で賄う構成を検討した。
「ドックが好き・前面スロット・LANが魅力」という要件があるためドック優先に戻した。
KVMは将来の選択肢として保留。
