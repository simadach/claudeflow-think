## 核心

Mac Mini と将来購入予定の Windows WS を、各1本のケーブルで共有デュアルディスプレイに接続する。モニターはAuto Input Detectionで自動切り替え。
**ケーブル構成要件**: 各 PC → ドック 1本・ドック → 各モニター 1本。分岐器・切り替え器などの中間機器は使わない。

---

## 背景・動機

- Mac Mini M4（メイン）+ Windows WS（予定）の2台体制に移行する
- 既存モニター LG 32UL750-W が1台あり、もう1台追加してデュアル化したい
- デスクをケーブルで汚したくない。各コンピューターからの出力は1本に絞りたい
- モニター2台を物理的にリンク（デイジーチェーン等）させたい

## 既知の懸念・トレードオフ

1. **HDR品質の差**：32UL750-W（DisplayHDR 600）と32U720A-B（HDR10のみ）でHDR品質が揃わない。HDRは対応コンテンツを見るときだけONにする運用のため、常時の表示差は問題にならないと判断。

2. **Auto Input Detection依存**：切り替えはモニター側機能に依存。応答速度は実機確認が必要。AIDの切り替えには5〜15秒かかり、2台のモニターが同時に切り替わらない場合がある。遅延が気になる場合は手動切り替えに変更する運用も想定。

3. **DP MSTハブ品質リスク（Windows側）**：安価品は4K/60Hz安定性にばらつきあり。BIOS/UEFIやロック画面はMST Hub非対応のため起動直後は片方のモニターにしか映らない場合がある。RTX 5070でDP×3確保できれば回避。

4. **Windows WS未購入**：GPUのポート構成により接続方式が変わる。WS購入後に確認が必要。

5. **両PC同時稼働時の意図しないAID切り替え**：Mac使用中にWindowsを起動すると映像信号をAIDが検出して切り替わる可能性。AID設定を「Manual」または「Last Input」に変更することで回避できる場合が多い。

6. **「1本縛り」とコストのトレードオフ（未解決）**：Mac Mini直結2本なら追加コスト0だが「1本縛り」を諦める。TB4/TB5ドック購入は¥49,990〜58,300の追加コストで「1本縛り」を達成する。この差額が体験価値に見合うかが未回答。

## 意識的に考慮したが却下した観点

**Thunderboltデイジーチェーン（1本で2台）**：
LG 32UL750-WにはThunderbolt入力がないため非対応と判明。断念。

**32UL950-W / 32UN650-W / 32UN950-W 中古**：
デザインを揃えるため検討。中古市場調査で在庫がほぼ存在せず断念。

**Dell U2723DE（Thunderbolt hub monitor）×2**：
TB4 + DP MST両対応だが¥80,000〜100,000/台で予算超過。断念。

**32U631A-B**：
安価（¥30,250）だが4K非対応（WQHD）・USB-C PD 15Wのみで不十分。

**A83B35A1（DisplayLink）**：
見た目が最も好み・LAN・SDカード・本体ディスプレイが魅力的だったが、
DisplayLinkに起因するHDCP制限（Netflix等の再生不可）・スリープ復帰問題・
常駐アプリ必須という継続的な摩擦リスクを嫌い、DisplayLinkを選択肢から完全除外した。

**A83B6 Prime（Dual Display 160W、¥29,990）**：
DisplayLinkなし・LAN搭載・¥29,990と魅力的だったが、
映像出力がDP Alt Mode → MST分岐 → HDMI×2という方式のため、
macOSがMST非対応であることが判明。Mac Mini M4では1画面しか映らない（またはミラーリングのみ）。
→ 完全除外。macOS環境では使い物にならない。

**Anker Prime A83B5（TB5、¥49,990）と1本縛りの費用対効果**：
TB5ドックはMac Mini M4（TB4）と後方互換で動作し、DisplayLinkなしでデュアル4K可能。
LAN・USB-A・SDカードも充実。ただし¥49,990は高価で、TB5の帯域はMac Mini M4では活かせない。
「Mac Miniから直接2本接続すれば¥0」という事実と比較して、¥49,990の価値があるかが未回答。
→ 査読に回して盲点を確認する。

**KVMスイッチ案**：
Mac・Windowsの切り替えをKVMスイッチ1台で賄う構成を検討した。
「ドックが好き・前面スロット・LANが魅力」という要件があるためドック優先に戻した。
KVMは将来の選択肢として保留。

## 🔁 査読で詰めたい論点：DisplayLink 再検討

**2026-06-25 追記**

DisplayLinkを一度除外したが、改めて「本当に除外すべきか」を査読で掘り下げてほしい。

### 除外した理由（再確認）

- HDCP制限：Netflix・Amazon Prime・Apple TV+等のDRMコンテンツが映らない
- 常駐アプリ（DisplayLink Manager）のインストールが必須
- スリープ復帰時にモニターが映らなくなる不具合報告
- macOSアップデートで一時的に動作しなくなるリスク

### DisplayLink再検討の動機

- DisplayLinkを除外するとTB4/TB5ドックしか選択肢がなく、最安でも¥49,990〜
- A83B35A1（DisplayLink搭載・¥38,709）は見た目が最も好みで、LAN・SDカード・本体ディスプレイも完備
- Mac MiniはデスクトップPCであり、MacBookと異なりスリープ頻度が低い（スリープ問題の影響が小さい可能性）
- Netflix等のストリーミングをMacのメインディスプレイで見ることはあるが、サブディスプレイで見るケースは少ない
- 「DisplayLink接続中のモニターがあるだけでMacBook内蔵ディスプレイでもDRM再生不可」という問題は、Mac Miniには内蔵ディスプレイが存在しないため影響範囲が異なるかもしれない

### 査読で確認してほしいこと

1. **Mac DesktopにおけるDisplayLinkのスリープ問題の実態**  
   MacBookで報告されるスリープ復帰問題は、Mac Mini（常時起動に近い運用）では発生しにくいか？

2. **HDCP制限の実際の影響範囲**  
   「DisplayLink接続モニターがある状態でMac Mini本体から映像出力するモニター（非DisplayLink）でもDRMコンテンツが映らなくなるか？」  
   → もし非DisplayLinkモニターは影響を受けないなら、Monitor 1（直接接続）でNetflixを見ればよく、運用で回避できる可能性がある
